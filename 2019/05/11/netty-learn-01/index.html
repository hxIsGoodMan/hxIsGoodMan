二、 NIO 核心类 介绍

Java NIO 由以下几个核心部分组成：

  • Channel
  • Buffer
  • Selector

Channel (通道)

是信息传输的通道，是java对输入输入的另一种抽象，可以类比BIO中的流的概念。而Channel的特性有三点：

  1. Channel是双向性的，流 分有 输入流 \输 出流。
  2. Channel 是非阻塞的。
  3. Channel 只能通过Buffer来操作。

Channel的核心实现类 ：
文件类： FileChannel
UDP类：DataGramChannel
TCP类： ServerSocketChannel \ SocketChannel

Buffer (缓冲区)

Buffer的作用是 操作(读写)Channel中的数据，它的本质其实是一块内存区域。

  • Buffer的核心属性
    1. Capacity (容量) ： 当前Buffer 的容量。
    2. Position (位置) : 有些许索引(index)的意思。也就是记录当前的位置，默认为0，最大为 capacity-1。
    3. Limit (上限) ： 写模式下 Limit 等同于 Capacity，读模式下 表示最多可以往Buffer中读多少数据，此时Limit会被设置成写模式下的 Position值。
    4. Mark (标记) ： 就是一个特殊的Position值，下次操作的时候 可以 回到Mark的位置 继续操作数据 。
  • Buffer的常用类
    • ByteBuffer
    • CharBuffer
    • DoubleBuffer
    • FloatBuffer
    • IntBuffer
    • LongBuffer
    • ShortBuffer

Selector （选择器\多路复用器）

selector 是NIO中 能够检测 多个NIO通道，并能知晓通道是否为注入读写事件做好准备的组件。通过它，一个单独的线程可以管理多个Channel，从而管理多个网络链接。
