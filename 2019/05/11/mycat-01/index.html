什么是MYCAT

  • 一个彻底开源的，面向企业应用开发的大数据库集群
  • 支持事务、ACID、可以替代MySQL的加强版数据库
  • 一个可以视为MySQL集群的企业级数据库，用来替代昂贵的Oracle集群
  • 一个融合内存缓存技术、NoSQL技术、HDFS大数据的新型SQL Server
  • 结合传统数据库和新型分布式数据仓库的新一代企业级数据库产品
  • 一个新颖的数据库中间件产品

MYCAT的前世今生

MYCAT 是 基于阿里开源的Cobar产品而研发，Cobar的稳定性、可靠性、优秀的架构和性能以及众多成熟的使用案例使得MYCAT一开始就拥有一个很好的起点，站在巨人的肩膀上，我们能看到更远。业界优秀的开源项目和创新思路被广泛融入到MYCAT的基因中，使得MYCAT在很多方面都领先于目前其他一些同类的开源项目，甚至超越某些商业产品。

MYCAT下载及目录解析

祭上 MYCAT 官网 http://www.mycat.io/

我下载的是 Mycat-server-1.6-release 时尚弄潮儿们，可以选择最新版的进行下载。 而我偏向稳定版

MYCAT 提供 linux版，windows版本，还有源码版。 业界良心啊有没有， 喜欢研究的同学可以下载源码好好专业一番。

我下载的是windows版。

conf 主要配置文件介绍

文件 说明
server.xml myCat配置信息 例如 账号，端口号等
schema.xml 数据库、表信息
rule.xml Mycat分片（分库分表）规则

上干货！！

使用MYCAT配置根据枚举分片

下面我们来做一个小例子 —— 使用mycat 搭建 mysql 集群 根据某个字段 分库存储信息

配置分片规则

分片规则 在rule.xml 中配置。
重点记下 tableRule 的 name (在schema.xml 中 需要用到)

增加my-rule.txt

配置物理数据库、表信息

上 schema.xml ！！

配置信息详解：
<table/> 这里配置了一个表 名为 customer ， 有两个数据库做分片 dn1,dn2 ，分片规则为 my-rule <dataNode/> 配置了两个 数据库， 数据库服务器信息 dataHost ，数据库信息 database ，节点名称 name
*<dataHost/> 配置了 物理数据库的一些信息 。这里配置了两个数据库服务器 做到读写分离，若只有一个数据库服务器只配置一个 <writeHost/> 即可

配置 MYCAT 账号信息

账号信息配置在 server.xml文件中

  • 这里配置了两个账号 一个root，一个user， user为只读账号。
  • server.xml 还可以配置 端口号，管理端口号，防火墙等信息，默认配置文件中的注释很齐全，有需要的可以去研究一下。

启动MYCAT

ok~~ 配置好后 就需要启动MYCAT了， 因为MYCAT 是基于java开发的，所以启动MYCAT之前请准备好java环境。

启动MYCAT需要进入bin目录：

  • windows下 进入cmd 执行 startup_nowrap.bat
    MyCAT Server startup successfully. see logs in logs/mycat.log 看到它，你就成功了

    linux下：

    • 启动 ./mycat start
    • 查看启动状态 ./mycat status
    • 停止 ./mycat stop
    • 重启 - ./mycat restart

    Mycat-server is running (20343). 看到它，你就成功了

    建表语句

    建好对应的库表之后 使用navicat连接 MYCAT (默认连接端口号为：8066)

然后再MYCAT中插入一条语句
INSERT INTO customer (id,name,rule) value(2,'吴彦祖','db2')
再查看db2 中

那你的MYCAT配置根据枚举分片 就完成啦~~
